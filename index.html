Introduction

Introduction

C is a widely used programming language developed in the early 1970s at Bell Laboratories. It is a by-product of UNIX operating system. The language was built by Ken Thompson and Dennis Ritchie in order to fulfill the need of having a high level language for the further development of UNIX OS. The first book on C is "The C Programming Language" by Brian Kernighan and Dennis Ritchie (1978). By 1980, C compilers were available on variety of machines running under different operating systems.


    Key Features

  • Low-Level Language

    • C is suitable for systems programming. It provides access to machine-level concepts (e.g. bytes and addresses) and operations that are closely to a computer's built-in instructions; hence programs can be fast.

  • Small Language

    • C provides a limited set of features than many languages. It relies on a library of standard functions.


    Strengths

  • Efficiency

    • - Because C was intended for applications where assembly language had traditionally been used, it was crucial that C programs could run quickly and in limited amounts of memory.

  • Portability

    • - C programs can run on computers ranging from PCs to super computers.

  • Power

    • - C language has large collection of data types and operators. As a result of this, we can accomplish task with a few lines of code.

  • Flexibility

    • - Although C was initially designed for system programming, C is now used for applications of all kinds (e.g. embedded systems, commercial applications). Few restrictions are on the use of its features (e.g. adding an integer value to a string).

  • Standard library

    • - contains hundreds of functions for input/output, string handling, storage allocation and other useful operations.


    Weaknesses

  • C programs are error-prone

    • - C's flexibility makes it an error-prone language. Programming mistakes that would be caught by other programming languages can't detect by a C compiler.

  • C programs can be difficult to understand

    • - C has some features that are not found in other programming languages. These features allow programmers to write programs in variety of ways; many of which obvious to original authors but not others.

  • C programs can be difficult to modify

    • - Large programs written in C can be hard to change if they haven't been designed with maintenance in mind.


First C Program

The C language facilitates a structured and disciplined approach to computer programming. C programming consists of one or more modules called functions. The following program displays the message "Hello world!" each time it runs. 

        Line 1------------>  #include 
        Line 2------------>  int main ()
        Line 3------------>  /* start of main */
        Line 4------------>  printf("Welcome to C Programming\n");
        Line 5------------>  /* program body */
        Line 6------------>  return 0;
        Line 7------------>  /* end of main */

Line 1:

"#include " Tells the C compiler to include the standard input output (1/0) header file stdio.h as part of the program, because it contains information that the C compiler needs to compile most programs. We must include in our program before "main".

Line 2:

The program begins with int main (). Many compilers require naming the function where the execution begins as main. The program's executable codes goes inside main() function. Following main there must be open and closed parentheses.

Line 3:

The open brace indicates to C, the beginning of the main function in the program.

Line 4:

To output information onto the screen, we use the word (function from standard I/O library) 'printf". It can display any type of variable formatted as a string enclosed in speech marks. "\n" tells printf() to advance to the next line after printing the message.

Line 5:

Represents the body (program statements) of the main function that performs the purpose of it. The lines between beginning and ending lines are the heart of a program. They are called statements. It is a complete instruction to a computer. The semicolon marks the end of the statement. There's no need of each statement fits on exactly one line. But as a good programming practice there should be no more than one statement per line in a program.

Line 6:

"return 0;" indicates that the program "returns" the value 0 to the operating system when it terminates.

Line 7:

The closed brace indicates the end of the main function. Everything between the braces is a part of the main function.
